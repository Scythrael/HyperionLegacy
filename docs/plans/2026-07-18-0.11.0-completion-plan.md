## Phase C: Salvage engine (live-only actions)

### Task C1: Equipment recycle-salvage

**Files:** NEW `src/lib/game/salvage.ts` (`salvageEquipment(state, instanceId, rng)` returns new state + a result summary), `salvage.test.ts`.
- Consume a SPARE crafted system; return a VARIABLE percentage (band ~30-40%, tunable consts) of its blueprint `recipe.inputs` to inventory (rounded sensibly, deposited via the quality-bucketed `addItemQuality`); free the slot. Reject fitted systems + Standard-Issue baselines (they have no blueprint / are managed).
- `rng` param defaults to Math.random; a seeded rng is injectable FOR TESTS ONLY (this is not an economy-tick seam).
- **Tests:** returns materials in the expected band (seeded rng); consumes the instance; frees a cap slot; rejects fitted + baseline pieces with a clear reason.
- **Commit.**

### Task C2: Salvaged-material loot roll

**Files:** `src/lib/game/salvage.ts` (`salvageSalvagedMaterial(state, itemId, rng)`), a loot-pool definition in model.ts (Task C3), `salvage.test.ts`.
- Consume one salvaged-material item; roll a weighted TIERED loot table (tiers named per gear rarity: standard/augmented/stellar/radiant + reserved above), reusing the mission-loot roll shape. Progression-gated ceiling: only tiers up to the player's current ceiling can roll (ceiling from progression + FA talent, Task C4). Deposit the rolled item at a quality appropriate to the tier.
- Balance: super-rare odds on plain refined/components; exclusive items weight the top tiers; the common outcome is modest.
- **Tests:** weighted tiers behave under seeded rng; progression gates the ceiling; super-rare refined/component odds; exclusive items only at high tiers; consumes the input.
- **Commit.**

### Task C3: Loot-pool data + exclusive items wiring

**Files:** `src/lib/game/model.ts` (the Damaged Reactor Housing's loot pool referencing A3's exclusive items + super-rare refined/components), tests.
- Define the per-salvaged-material loot pool (a tiered table). Reference the exclusive items from A3.
- **Tests:** the pool is well-formed (every referenced item exists; tiers ascend; exclusives present at top).
- **Commit.**

### Task C4: FA salvage talent

**Files:** the FA talent-tree data (find it: model.ts talent definitions) + wire the talent's effect into `salvageEquipment` (yield %) and `salvageSalvagedMaterial` (odds / ceiling), tests.
- ONE combined talent (recycle yield + loot odds/ceiling), first-pass values. Slots into the existing radial talent web data.
- **Tests:** the talent raises the recycle yield band and the loot ceiling/odds when learned.
- **Commit.**
