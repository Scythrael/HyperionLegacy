## P1: Theme completeness (starfield + log text follow the theme)

Today the ambient Starfield dots (`#bfe9f5`, `Starfield.svelte`) and the log text (`#9fc4cc`, `App.svelte` `.log-entry`) are hardcoded and do not recolor with the theme, so theme switching looks slightly unfinished.

- Add a `--color-starfield` token to `:root` and to every `[data-theme=...]` block in `app.css`. Each is a near-white star color with a SUBTLE hint of that theme's accent (default `[data-theme` cyan stays today's `#bfe9f5`). Keep it subtle: this is a background, it must not compete with content.
- `Starfield.svelte` `.star` background: `var(--color-starfield)` instead of the hardcoded hex.
- Point `.log-entry` color at the existing per-theme `var(--color-text-secondary)` (or a new `--color-log` if a distinct shade reads better) so log text shifts with the theme.

**IMPORTANT forward constraint (user, 2026-07-17):** this theme tint applies ONLY to the DEFAULT ambient starfield. The FUTURE selectable background styles (sub-light stars streaking toward the player, warp-jump streaks; see SUGGESTIONS.md "Selectable background styles") must keep their own PROPER fixed colors and must NOT theme-shift. Whatever implements `--color-starfield` must leave each future style free to set its own colors, not impose the theme tint globally. Add a comment saying so where the token is defined and where Starfield uses it.

## P3: Modal a11y (focus-trap + Escape + aria across the three modals)

Three modals share the `.modal-backdrop` / `Panel.modal-dialog` pattern with the same gaps (KNOWN_ISSUES.md): delete-save (`deleteModalOpen`), captain-select (Fleet Operations popup), import-save (`importModalOpen`). None trap focus, none close on Escape; the captain-select modal has zero aria-labels.

- Build ONE shared mechanism (a Svelte `use:` action, e.g. `src/lib/focusTrap.ts`): on mount, move focus into the dialog; trap Tab/Shift+Tab within it; close on Escape (via a callback); restore focus to the previously-focused element on destroy.
- Apply it to all three modals. Add `role="dialog"`, `aria-modal="true"`, and an `aria-label`/`aria-labelledby` to each (the captain-select one especially).
- Escape calls the same close path as the Cancel button. Do NOT change the typed-"DELETE" gate behavior of the delete modal.

## P4: Corrupt-save recovery (do not silently wipe an unloadable save)

Today `loadFromLocalStorage()` (`App.svelte:1048`) returns null when `deserialize` fails, and the game silently starts fresh, so a corrupt save is lost with no warning.

- Distinguish "no save at all" (fine, fresh start) from "a save EXISTS in localStorage but failed to deserialize" (corrupt). Add a save.ts helper that reports which case it is without discarding the raw string (e.g. return a typed result, or expose `hasRawSave()` + keep `exportRawSave()` working on the corrupt raw).
- On corrupt: do NOT overwrite or clear it. Show a recovery modal: explain the save could not be loaded, show the raw save text in a copyable field (reuse `exportRawSave`/`downloadRawSave`), and let the player choose to download a backup and then start fresh (an explicit action), rather than an automatic wipe.
- Use the P3 focus-trap for this modal too.
