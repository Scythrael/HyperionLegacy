# Equipment 0.11.0 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (or superpowers:subagent-driven-development) to implement this plan task-by-task.

**Goal:** Ship the non-combat equipment slice: craft, fit, and see the stat impact of gear on the 4
live slots across the 4 existing hulls, with the item-generation engine, Quality axis, cargo-as-true-cap
redesign, storage, and migration, and the full stat vocabulary (incl. 9 combat weapons) defined in data
so 0.12.0 plugs in without a rewrite.

**Architecture:** Additive to the existing closed-form engine. New pooled `EquipmentInstance[]` on
`GameState` (single-source-of-truth fitment pointer, mirrors `ShipInstance`). A pure item-generation
pipeline (iLevel -> budget -> ratio -> weighted affix roll, seeded for offline parity). Equipment stats
fold into `shipDerivedStats` BEFORE `effectiveMissionDef`, stable per cycle (fitting locked mid-mission),
so `tickCaptainMission` stays closed-form. Quality is a universal 0-5 axis retrofitted onto materials/
refine/fabricate.

**Tech Stack:** Vite + Svelte 5 + TypeScript, Decimal (big.js-style) for idle-scale numbers, Vitest.
Node not on PATH: prefix every command with `export PATH="/c/Program Files/nodejs:$PATH"`. Gate each task
with `npm run check` (expect 0 ERRORS) and `npm test`.

**Design docs:** `2026-07-17-equipment-0.11.0-design.md` (slice), `2026-07-17-ship-equipment-combat-epic-design.md` (vocabulary).

**Key files to orient on before starting:** `src/lib/game/model.ts` (types + tables: `ItemDef`,
`BLUEPRINTS`, `ITEMS`, `SHIP_TYPES`, `ShipInstance`, `ShipDerivedStats`, `effectiveMissionDef`),

## Phase 2: Item-generation engine (pure, seeded)

### Task 4: iLevel calculation

**Files:** Create `src/lib/game/itemgen.ts`; Test `src/lib/game/itemgen.test.ts`.

```ts
// iLevel is additive and modest, capped by the item's tier. Big numbers come from the budget
// conversion (Task 5), never from stacking multipliers into iLevel itself.
export function computeItemLevel(args: {
  craftingLevel: number;
  achievementBoost: number;   // flat, +5 per relevant achievement (0 for now)
  faTalentBonus: number;      // optional, 0 for now
  itemTierCap: number;        // the recipe/item tier ceiling
}): number {
  const raw = args.craftingLevel + args.achievementBoost + args.faTalentBonus;
  return Math.min(raw, args.itemTierCap);
}
```

**TDD:** tests for below-cap, at-cap, above-cap clamp, and additive contribution of boosts. Commit.

### Task 5: Budget pipeline (compounding, deterministic)

**Files:** Modify `src/lib/game/itemgen.ts`; Test `itemgen.test.ts`.

```ts
// All multipliers are TUNABLE constants (balanced later via the play-sim). Budget COMPOUNDS quality
// then rarity (0.11.0 caps at Radiant; ascension terms exist but never fire this patch).
export const PER_LEVEL_BUDGET = 2;        // TUNABLE
export const QUALITY_MULT = 1.1;          // TUNABLE, per quality rank
export const RARITY_MULT = 1.15;          // TUNABLE, per base-rarity index
export function computeBudget(iLevel: number, quality: number, rarityIdx: number): number {
  const base = iLevel * PER_LEVEL_BUDGET;
  const withQuality = base * Math.pow(QUALITY_MULT, quality);         // quality 0..5
  const withRarity = withQuality * Math.pow(RARITY_MULT, rarityIdx);  // Standard(1)..Radiant(4)
  return Math.round(withQuality * 0 + withRarity); // round up handled by callers; see note
}
```

**TDD:** reproduce the design doc worked example within rounding (ilvl feeding through quality 5 and
Radiant), plus monotonicity (more quality/rarity never lowers budget). Commit.
