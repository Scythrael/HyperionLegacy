# Equipment 0.11.0: Buildable Slice Design

**Date:** 2026-07-17
**Version:** ships as `0.11.0` (APP_VERSION bump on start). One complete patch, built layer by layer
with staging device-tests at each step, shipped whole (no `.1`/`.2` stub releases).
**Save impact:** migration required (equipment arrays, Crafting Level, item-instance + material-quality
fields, ship equipment fitment). SAVE_VERSION bumps.
**Parent vocabulary:** `2026-07-17-ship-equipment-combat-epic-design.md`.

Strawman numbers below are FIRST-PASS and TUNABLE; red-pen freely. Per-slot stats/affix pools are
proposals for review (the B6 promise).

---

## 1. What ships in 0.11.0 (the endpoint)

A player can, end to end: gather -> refine -> fabricate -> build a ship -> craft equipment -> pick a
captain's ship -> fit it out, swap and discard gear -> see the stat deltas with percents on the captain's
status card. End state: a captain in a freshly built ship with a full gear set. That is the definition
of done, demonstrated on the 4 hulls that exist (Freighter + 3 Prospector).

**In scope:**
- Equipment **instance model** + **affix/item-generation engine** (Section 3), producing Standard-Radiant
  gear with quality 0-5.
- **Crafting Level + iLevel** (Section 3a).
- The **4 live slots** with implicits + affix pools + 3 spec-quirk varieties + Standard-Issue baselines
  (Section 2).
- **Fitting system** (equip/unequip, atomic swap, on-mission lock, closed-form fold-in via both tick
  paths) (Section 4).
- **Quality (0-5) axis** across materials/components/gear + material rarity/quality + prospector-build
  influence on drops (Section 5).
- **Cargo-as-true-cap redesign** (Section 6).
- **Equipment storage tab** (25 cap, upgradeable) + discard/sell/salvage (Section 7).
- **Reactor Power budget** (present, non-binding) + **Mass budget** (binds via Fuel Efficiency + FTL
  Speed, which are live) (Section 8).
- **Migration** (Section 9) and **mockup-gated UI** (Section 10).
- The **full stat vocabulary defined in data** (including combat stats + the 9 weapons' stat shapes) so
  0.12.0 plugs in without a data-model rewrite. Combat stats exist as fields; nothing consumes them yet.

**Explicitly NOT in scope (defined-and-reserved):** any weapon/shield/hull/thruster/cockpit/quarters/
sensor gear (combat/crew slots, 0.12.0), rarities above Radiant (legendary/set/ascension), durability
LOSS (no combat), modules, drops, crew, exploration.

## 2. The 4 live slots

Each slot: guaranteed **implicit** line(s) + rolled affixes from a per-item WEIGHTED pool. Each slot has
**3 varieties** (same core job, spec-favoring quirks). Standard-Issue = the quality-0 baseline version,
auto-fitted, keeps a bare ship dispatchable.

### Cargo Bay (universal)
- **Implicit:** Cargo Capacity.
- **Varieties (3):** Prospector Hold (90% cargo / 10% drone-cap-reserved, favors extraction runs),
  Balanced Hold, Hauler Hold (max raw cargo, more Mass).
- **Affix pool (strawman):** +Cargo Capacity, -Mass (lightweight framing), +Fuel Efficiency (aero
  stowage), +Extractor Efficiency (quick-stow). Weighted so +Cargo is common, cross-stat affixes rare.
- Interacts with the cargo-as-true-cap redesign (Section 6): its Cargo Capacity raises the real hold cap.

### FTL Drive (universal)
- **Implicit:** FTL Speed + Fuel Efficiency (two implicits).
- **Varieties (3):** Sprint Drive (FTL Speed heavy), Economy Drive (Fuel Efficiency heavy), Balanced
  Drive.
- **Affix pool:** +FTL Speed, +Fuel Efficiency, +Fuel Capacity, -Mass.

### Reactor Core (universal)
- **Implicit:** Power Output.
- **Varieties (3):** High-Output Core (more Power, more Mass), Efficient Core (less draw headroom but
  light), Balanced Core.
- **Affix pool:** +Power Output, -Mass, +Fuel Efficiency (reactor feeds drives), -Power Draw-of-fitted
  (efficiency affix, reserved usefulness until 0.12.0 high-draw gear).
- Power budget is present but non-binding in 0.11.0 (low-draw gear).

### Spec Utility Slot: Prospecting Rig (Prospector hulls only)
- **Implicit:** Extractor Efficiency.
- **Equip requirement:** prospecting-spec captain in a Prospector hull (the general requirement-gating
  system; see epic Section 1).
- **Varieties (3):** Yield Rig (Extractor Efficiency heavy), Survey Rig (Sensors: faster find / material
  rarity chance), Refinery-Feed Rig (favors quality of found materials).
- **Affix pool:** +Extractor Efficiency, +Sensors (find speed + rarity chance), +material quality chance,
  -Mass.
- This is the one Spec Utility item shipping in 0.11.0 (other specs' hulls do not exist yet).

Freighter has 3 universal slots (Cargo, FTL, Reactor), no Spec Utility. Prospector hulls have all 4.

**Reserved slots on every ship (defined, empty, no craftable gear in 0.11.0):** Cockpit, Quarters,
Thrusters, Sensor, Shield Emitters, Hull Plating, Propellant Tanks, Weapon hardpoints, module slots.

## 3. Item-generation engine (0.11.0 subset)

Pipeline from the epic Section 5, reduced to what 0.11.0 produces (quality 0-5, rarity Standard-Radiant,
no ascension/sets, durability stored but never lost):

`budget = (iLevel x perLevelBudget) x qualityMult(0-5) x rarityMult(Standard-Radiant)` ->
distribute by per-item ratio -> roll each line from the weighted affix pool.

- Crafted rarity rolls Standard-Radiant at base; Constellar/Luminous/Nova require crafting talents that
  proc (talents exist as reserved nodes; the proc path is defined so the data supports it, but the
  higher-rarity CONTENT is not built).
- Instance shape (`EquipmentInstance`): id (from a `nextEquipmentId` counter), slotType, quality (0-5),
  rarity (base enum + ascension:none), blueprintKey, implicitStats, rolledStats, mass, powerDraw,
  durability (max + current, current never drops in 0.11.0), fittedToShipId (null = spare), reserved
  `integrity`. Global pool `GameState.equipment: EquipmentInstance[]`.

### 3a. Crafting Level + iLevel
- New `GameState.craftingLevel` + `craftingXp`. Every completed refine/fabricate/ship-build/equipment
  craft grants crafting XP scaled to the item's tier/cost.
- `iLevel = min(craftingLevel + achievementBoost + optionalFAtalentBonus, itemTierCap)`, additive.
- Achievement +5 boosts and the FA talent are reserved hooks (a couple can ship if easy; not required).

## 4. Fitting system (closed-form critical)

- Equip/unequip is an **atomic swap** (like ship assignment): fitting new gear moves the old piece back
  to the storage pool; a live slot is never empty (Standard-Issue re-fits if you unequip to nothing).
- **Cannot change fitment while the captain is on an active mission** (mirrors the ship-swap lock), which
  is what keeps `effectiveMissionDef` stable per cycle.
- Equipment stat contributions fold into `shipDerivedStats` BEFORE `effectiveMissionDef`. Because
  fitment is stable per cycle, the one-big-call == many-small-calls guarantee holds; no engine-loop
  change.
- **MUST wire equipment stats into both `tick()` and the live `App.svelte` loop (or the shared helper),
  with an offline-parity test.** This is the recurring "live loop drifted from tick()" crack; ship stats
  fell through it once (fixed in `9fc67a6`).

## 5. Quality axis + material rarity/quality

- Quality is a universal 0-5 value shown as icons; rolled at every stage (gather, refine, fabricate,
  build). Rolls compound and are rare (placeholder ~1/1000/tier, TUNABLE hard against gather volume;
  higher quality may reduce craft material cost).
- Materials carry BOTH rarity (caps craftable gear rarity) and quality (feeds craft quality + the
  rarity-upgrade odds). Missions drop escalating material rarity/quality over time.
- Prospector build influences drops: Sensors -> rarity chance + faster find; Extractor Efficiency ->
  quality; separate roll -> quantity.
- This retrofits the Warehouse (store per quality), Refinery (input quality -> output quality), and
  Fabricator (input quality -> craft quality). It is the biggest existing-system touch in 0.11.0; build
  it as a discrete tested layer.

## 6. Cargo-as-true-cap redesign (folded in)

Fixes the shipped 94-over-90 behavior (cargo drives extraction LENGTH, not a return cap). Closed-form
safe approach (from the logged brainstorm): decouple the mission's guaranteed haul from the ship's Cargo
Capacity (a real hold cap with headroom), then CLAMP total returned cargo at the cap. Never a
stop-when-full RNG timer (that breaks closed-form). Then Cargo Bay gear cleanly raises the cap. Extend
the closed-form tests to cover the clamp.

---

## 7. Equipment storage + management

- New Warehouse **equipment tab**: slot-capped (start 25, upgradeable), holds only UNFITTED spares
  (equipped gear lives on the ship, off the tab; unequipping returns it).
- Actions: **discard** (delete), **sell** (credits), **salvage** (~60% of the craft materials back).
- Route sell/salvage credits + material returns through the existing economy; salvage returns through
  the allocation model.

---

## 8. Budgets

- **Mass:** every item has intrinsic Mass; ship total Mass penalizes Fuel Efficiency + FTL Speed (both
  live in 0.11.0), so Mass is meaningful now. Penalty is a mass-vs-thrust ratio (TUNABLE), offset by
  propulsion stats. A -Mass affix is desirable.
- **Power:** every item has Power Draw; Reactor Core provides Power Output; total draw must fit Output.
  Present and honest in 0.11.0 but rarely binds (low-draw non-combat gear). Binds hard in 0.12.0.

## 9. Save migration

- Add `GameState.equipment: EquipmentInstance[]`, `nextEquipmentId`, `craftingLevel`, `craftingXp`, per-
  material quality fields, and per-ship equipment fitment.
- Seed **Standard-Issue gear** into every existing ship's live equipment slots (so migrated ships stay
  dispatchable, mirroring how the ships-foundation migration seeded a Freighter per captain).
- Existing materials get quality 0. New game seeds the same shape. Frozen-migration discipline: new
  numbered entry only, never edit shipped bodies.

## 11. Testing + dev loop

- **Dev loop:** build a layer -> `npm run check` (0 errors) + `npm test` -> push to staging -> user
  device-tests -> next layer. Ship the whole thing to prod as 0.11.0 only after the full-pipeline
  playtest (gather -> ... -> fit -> see stat deltas) passes on a real device + a real migrated save.
- **High-risk seams to controller-verify personally:** the closed-form fitting fold-in (both tick paths
  + offline parity), the affix-roll offline parity, the cargo-as-true-cap clamp parity, and the
  migration seeding Standard-Issue onto a real pre-0.11.0 save.
- Authored unit tests for: the item-gen pipeline (budget -> ratio -> weighted roll determinism at a
  seed), quality/rarity gating, fitting swap + on-mission lock, cargo clamp, migration shape.
