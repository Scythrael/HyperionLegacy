# 0.11.1 "UI Restructure" notes (storage architecture + Quartermaster + Help)

**Status:** NOTES for the NEXT patch after 0.11.0. Captured 2026-07-18. The USER APPROVED shipping 0.11.0's Ship Systems + Salvaged Materials tabs inside the EXISTING Warehouse ON THE EXPLICIT CONDITION that this restructure happens correctly in 0.11.1. This doc is that promise. Do NOT let it slip.

## The storage-architecture split (the core of 0.11.1)

Today (0.11.0) everything shares one Warehouse. 0.11.1 splits storage into TWO homes:

**1. Warehouse** keeps RAW inputs and intermediates only:
- Raw materials, Refined materials, Fabricated components, and Salvaged materials.
- Each inventory tab gets PROPER SUB-CATEGORIES (not one flat grid): e.g. Raw -> Ores, Organic Compounds, (etc.); Refined and Components similarly sub-categorized. Still split by TIER. Goal: easier to track.

**2. Quartermaster (working name) = a NEW facility for FINISHED / equippable / consumable products:**
- Tabs: **Ship Systems**, **Ship Weapons**, **Ship Modules**, **Consumables**, and eventually **crew weapons / armor / gear**.
- Each tab split by TIER and by CATEGORY, same tiled style as the Warehouse.
- Has its OWN **Overview** and **Upgrades** tabs (separate from the item tabs).
- **The 0.11.0 Ship Systems + Salvaged Materials tabs MOVE here in 0.11.1** (they live in the Warehouse only as the interim 0.11.0 home). This move is the user's approval condition.

## Salvaging as its own facility (up for debate)

The user leans toward salvaging being SEPARATE from refining and fabrication, possibly its own facility. Decide during 0.11.1 design. The 0.11.0 salvage actions (recycle + loot roll) already exist as pure functions (`salvage.ts`); 0.11.1 just gives them their proper home/surface.

## Deferred FURTHER to 0.12.0 (combat), NOT 0.11.1

- **All slots get IMPLICITS based on the slot itself**, including the current systems. The user deferred this to the combat pass because many implicits arrive WITH the new combat slots (shields, hull plating, weapons), so it is cleaner to define the whole implicit set at once. 0.11.0 leaves current systems with whatever implicits they have now.
- The equipment TOOLTIP (built in 0.11.0) is structured to hold weapon rows (damage range / DPS / fire rate) and SECONDARY effect-affixes (descriptive sentences like "Increases armor penetration by N"); those rows populate when combat adds weapons + effect-affixes. 0.11.0 renders only header + implicits + primary `+stat` affixes.
